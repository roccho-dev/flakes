# Current Goal Delta

Status: partial

## Meaning

Plan by comparing:

1. current state
2. target end-state
3. the explicit delta between them

This prevents local `how` from replacing the actual `what`.

## Rules

- restate the current state before major transitions
- restate the target end-state before major transitions
- derive tasks from the delta, not from habit or recent momentum
- periodically re-check assumptions, dependency anchors, and the real target end-state

## Existing Reflection

- `RUNBOOK.md` already requires state reconstruction after memory loss
- `RUNBOOK.md` already requires updated tables on every loop

## Missing Reflection

- no explicit requirement yet says to perform periodic reanalysis of assumptions,
  dependency anchors, and target end-state even when memory is intact

## Completion Target

The manage system should make `current -> goal -> delta` an explicit planning step,
not only an implicit operator habit.
