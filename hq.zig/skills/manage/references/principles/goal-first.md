# Goal First

Status: partial

## Meaning

Define the desired end-state before discussing methods.

`goal` should say what must be true when the scoped member is done.
It should not silently turn into a to-do list or implementation recipe.

## Rules

- express the expected completion state before asking for work
- keep `goal` declarative
- keep `status` declarative
- do not let local next-step planning replace the target end-state
