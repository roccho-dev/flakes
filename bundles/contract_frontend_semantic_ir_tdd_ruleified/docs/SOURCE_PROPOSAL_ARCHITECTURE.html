Recommendation

まずは frontend compile の意味保存を固定する

提案の芯は、現在の canonical strict core の手前に Semantic Contract IR を置くことです。契約フロントエンドは、目的・事象をいきなり backend へ寄せず、まず 集合意味 と 観測意味 に正規化します。

  • CUE は emitter の一例でしかない。
  • JSON Schema / CUE / SQL / Lean4 で共有したいのは、構文ではなく意味。
  • Datalog は grammar police ではなく、trace と obligation の番人に置く。
